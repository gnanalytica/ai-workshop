## ⏪ Pre-class · ~30 min

### Setup

- [ ] Capstone deployed and reachable (Vercel / Render / HF Space / ngrok backup).
- [ ] 5-slide deck (problem · who · live demo · what's next · ask) as PDF.
- [ ] **Backup screen-recording** of a successful run — non-negotiable.
- [ ] Charged laptop, charger, hotspot, water.

### Primer

- **Watch:** Past-cohort highlight reel — https://www.youtube.com/watch?v=sz_LgBAGYyo
- **Read:** YC's "How to demo." Internalise the "show, don't tell" rule.

### Bring to class

- [ ] One sentence: *"In 5 minutes you will see ___ go from ___ to ___."*
- [ ] Two anticipated tough questions and a one-line honest answer for each.

> 🧠 **Quick glossary.** **Cold open** = the first 15 seconds. **Live demo** = real input, real output, on stage. **Vapourware** = slides describing things that don't run. **Backup tape** = pre-recorded run, used only if live fails.

## 🎥 In-class · live session

### Running order

| Block | Time | What |
|---|---|---|
| Welcome + judging panel intro | 10 min | Sandeep + 3 judges |
| Demos 1–6 | 50 min | 5 min demo + 3 min Q&A each |
| Break | 10 min | Coffee, reset rooms |
| Demos 7–12 | 50 min |  |
| Judges huddle + announcements | 10 min | Day-29 highlight + tomorrow logistics |

### The 5-minute structure that works

1. **0:00–0:30** — cold open with a live action. *"Watch what happens when I paste this hostel-mess complaint."*
2. **0:30–1:00** — problem + who hurts.
3. **1:00–3:30** — live demo. **Two flows max.** Don't tour the UI.
4. **3:30–4:30** — what's under the hood (one architecture slide).
5. **4:30–5:00** — what's next + the one ask (users / data / role / co-founder).

### When things break on stage

- Stay flat. *"Live demos fail. Switch to recording."*
- Don't apologise twice. Move on.
- Q&A: *"I don't know — I'll find out by Friday"* beats every fake answer.

## 🧪 Demo slot · the actual run

This is where the lab section normally sits — today, this **is** the lab.

- Demoers: arrive 15 min before slot. Test screen-share. Mic check.
- Audience: silent during demos, type questions in the Q&A doc.
- Everyone: fill the peer-feedback form (link in dashboard) for every demo you watch.

**Artifact.** Your demo recording auto-saved to the cohort drive + judges' rubric scores returned by tomorrow.

> ⚠️ **No live coding fixes on stage.** If the bot breaks, switch tape. Code-fixing during a demo is the fastest way to lose the room.

## 📊 Live poll · audience feedback

**After each demo, audience votes:** Would I use this / Would I pay for this / Would I work on this / Would I invest / Not yet. Used to drive the post-demo discussion and the Day-30 leaderboard.
