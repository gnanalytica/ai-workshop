Anyone can ship a prompt. Engineers ship a prompt **with an eval set behind it**. Today is the day your capstone stops being a vibes-based demo.

## 🎯 Today's objective

**Topic.** Local LLMs, prompt patterns, eval habits, tracing.

**By end of class you will have:**
1. Llama-3.1-8B running locally via Ollama, answering your capstone's core prompt.
2. Three prompt patterns applied — *role + format + few-shot* — and the before/after delta measured.
3. A 10-case eval set running through OpenEvals, with traces visible in LangSmith.

> *Why this matters.* Demo Day judges will ask "how do you know it works?" "It looked good in testing" loses. "Here's our 10-case eval, 90% pass, here's the trace" wins.

> 🧠 **Quick glossary.** **Local LLM** = model running on your laptop (no API). **Eval** = a test case for an LLM. **Trace** = the full record of a run: inputs, prompts, tokens, latency. **OpenEvals** = LangChain's eval harness. **LangSmith** = the trace viewer.

### Three prompt patterns, in order of ROI

1. **Explicit role + output format.** *"You are X. Return JSON with keys A, B, C. No prose."* Cuts hallucination, makes parsing trivial.
2. **Few-shot with edge cases.** Three examples beats three paragraphs of instruction. Pick examples that look like the *failures* you've seen.
3. **Chain of thought / scratchpad.** *"Think step-by-step in `<thinking>` tags before you answer."* Worth the tokens for any reasoning task.

### When local Llama actually wins

- **Privacy** — student health data, exam answers, anything you can't send to OpenAI.
- **Cost at high volume** — once you cross ~10M tokens/month, your laptop or a cheap GPU box wins.
- **Offline / unreliable network** — campus during exams, rural deployments.
- **Latency** — first token in 50ms vs 800ms over the API.

It does *not* win on raw quality. Llama-3.1-8B is roughly GPT-3.5 level. Plan accordingly.

## 🧪 Lab: Local LLM + eval set + trace

1. Pick **one** capstone prompt. Run it through `ollama` directly: `ollama run llama3.1:8b "<your prompt>"`. Note the output.
2. Apply the three patterns (role/format, 2 few-shot examples, scratchpad). Run again. Note the delta.
3. Build `evals.jsonl` — 10 lines, each `&#123;"input": "...", "expected": "..."&#125;`. Indian-context examples preferred.
4. Wire OpenEvals: write a grader (`exact_match` or LLM-as-judge), run all 10 cases against both `llama3.1:8b` and `gpt-4o-mini` with `LANGCHAIN_TRACING_V2=true`.
5. Open LangSmith. Find the slowest case. Find the most expensive case. Screenshot both.

**Artifact.** Push `evals.jsonl`, `run_evals.py`, and a `README.md` with your pass-rate table + LangSmith share link to `/labs/day-25/`.

> ⚠️ **Indian-context note.** Llama-3.1 tokenizes Devanagari/Tamil at ~2 chars/token vs ~4 for English. Hinglish prompts cost more than they look. Measure.
