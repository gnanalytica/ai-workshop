A chatbot answers. An agent **acts**. The difference is a loop, a set of tools, and a controller that decides what to do next. Today we open that loop.

3. **Sketched** a 2-agent CrewAI team for your capstone (researcher + writer) and written one paragraph on when MCP beats a custom tool.

> *Why this matters.* The word *"agent"* is in every capstone pitch this cohort will hear. Today you go from hearing it to recognising one — and to knowing when NOT to build one.

> 🧠 **Quick glossary.**
> **ReAct** = the loop where the LLM **Re**asons and **Act**s in turns.
> **Tool** = a function the LLM is allowed to call (search, calculator, send-email).
> **Trace** = the printed log of every Thought → Action → Observation step.
> **LangChain** = the toolkit for building LLM apps.
> **LangGraph** = the agent runtime built on top — handles the loop.
> **MCP (Model Context Protocol)** = an open standard from Anthropic so the same tool can plug into Claude Desktop, Cursor, your own agent — no rewiring.
> **CrewAI** = library for hiring multiple agents (researcher + writer + critic) instead of one giant agent.

### The minimum agent

```
loop:
  thought  = LLM(state)         # "I need the latest GDP figure"
  action   = pick_tool(thought) # web_search("India GDP per capita 2025")
  observe  = run_tool(action)   # "$2,710 per IMF April 2025"
  state    = state + observe
  if LLM says "done": break
```

That's it. Everything else — LangGraph, CrewAI, AutoGen — is sugar on this loop.

### When to reach for what

- **One LLM call** — classification, summary. Don't build an agent.
- **Fixed pipeline (workflow)** — LangChain chain. Predictable, cheap, debuggable.
- **Agent (LangGraph)** — when the *path* depends on intermediate results.
- **Multi-agent (CrewAI)** — when roles are genuinely different and parallel: researcher + critic + writer.
- **MCP** — when the *same* tool needs to plug into Claude Desktop, Cursor, your custom agent, all at once.

4. **Sketch a 2-agent crew · 3 min.** On paper or in your M2 doc, sketch a CrewAI team for your capstone: one **researcher** agent + one **writer** agent. Note each agent's *role*, *goal*, and *tools*. No code needed.

**Artifact.** Three screenshots + the 2-agent sketch, all dropped into a single Google Doc named `Day 15 — Agent Lab — <Your Name>`. Submit on the dashboard.

### Bonus track — build it from scratch (optional, 25 min)

For students with a Python comfort:

1. New file `agent.py`. Define two `@tool`s — `web_search(query)` (use Tavily or DuckDuckGo) and `calculator(expression)`.
2. Build a LangGraph `StateGraph` with two nodes (`agent` and `tools`) and the loop `agent → tools → agent → END`.
3. Run with the prompt of the day. Print every `(thought, action, observation)` triple.
4. Break it on purpose: ask a question that needs **three** tool calls. Confirm the loop holds.

**Bonus artefact.** `agent.py` + a `trace.txt` in your repo at `/labs/day-15/`. Push. +3 bonus points.

> ⚠️ **Cost watch.** Each agent run costs roughly ₹1–4 in tokens. **Always** set `recursion_limit=10` before invoking — a runaway agent can quietly burn ₹500 in one bad loop. Last cohort had one. Don't be that one.
