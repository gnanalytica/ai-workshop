A webpage you can't share is just a file. Today you build one, push it to GitHub, and put it on the internet — using plain English and AI. No prior coding experience needed.

## 🎯 Today's objective

**Topic.** Vibe coding a static webpage and deploying it live with GitHub Pages.

**By end of class you will have:**
1. A fully animated static landing page for a fake product, running locally on your browser.
2. The same page live on a public URL via GitHub Pages — shareable with anyone.
3. Made at least one live change via Antigravity and re-deployed it.

> *Why this matters.* Every product starts with a webpage. Whether you are launching a startup, presenting a capstone, or just proving something exists — shipping a live URL in under 3 hours is a skill you will use forever.

### How the web actually works (mental model)

```
You type a URL
  → browser asks a server: "give me this page"
  → server sends back index.html + style.css + script.js
  → browser reads those files and draws the page

Static site: the server just sends files. No logic, no database.
GitHub Pages: GitHub IS the server. Your repo IS the files.
```

That's all of today in one diagram. Everything else is detail.

### The vibe coding workflow

```
1. Open Antigravity → new project
2. Create agents.md  → tells the AI what kind of project this is
3. Connect MCP plugins → GitHub MCP + Filesystem MCP
4. Type a prompt describing your page
5. AI writes all the code
6. Open index.html in browser → see it locally
7. Ask Antigravity to change something → refresh → see the update
8. Push to GitHub → enable Pages → live URL
```

### Setting up agents.md (show students exactly this)

Create a file called `agents.md` in your project root. This briefing file tells the AI the rules before it writes anything:

```markdown
# Project: My Landing Page

## What this is
A static animated landing page for a fictional product.
Built with plain HTML, CSS, and JavaScript only.
No frameworks. No npm. No build tools.

## Rules
- All code goes in a single index.html file
- Use CSS animations for all motion effects
- Keep it simple — must work by opening index.html in a browser
- Deploy target: GitHub Pages
```

### Connecting MCP plugins

In Antigravity settings, enable two plugins before building:

- **GitHub MCP** — lets the AI push code directly to your GitHub repo.
- **Filesystem MCP** — lets the AI read and write files on your computer.

> 🧠 **What is MCP?** Model Context Protocol — a standard way to give your AI extra abilities. Without GitHub MCP the AI can write code but can't push it. With it, it can do both. Think of MCPs as apps you install for your AI.

### Live demo — AirDosa animated landing page

Instructor builds "AirDosa" (AI-powered dosa delivery by drone) from scratch in real time using Antigravity. The full arc:

1. **Prompt** → page is generated
2. **Open index.html locally** → see it in browser
3. **Make a change via Antigravity** → refresh → change appears
4. **6 terminal commands** → pushed to GitHub
5. **3 clicks in GitHub Settings** → live on GitHub Pages
6. **One more change + push** → live URL updates → full loop shown

**The 6 deploy commands (students follow along):**

```bash
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/YOUR-REPO.git
git push -u origin main
```

**After first deploy, every future update is just:**

```bash
git add .
git commit -m "describe your change"
git push
```
