By Day 4 you can pick a model and prompt it well. By Day 5 you stop treating "AI" as one switch. The same question, asked the same way, looks very different in **Fast**, **Thinking**, and **Deep** — and once you can feel the difference, you can pick the right one for the job.

## 🎯 Today's objective

**Topic.** Grounded research with Gemini: Fast vs Thinking vs Deep.

**By the end of this class, you will:**

**1. Explore Different Models.** Understand how to use Gemini to analyze outputs from three different models: **Fast**, **Thinking**, and **Deep**.

**2. Utilize Prompting Skills.**
- Apply the prompting techniques you learned on Day 3 to enhance the quality of your outputs.
- Craft clear and specific prompts to guide the models effectively.
- Experiment with different phrasing to see how it affects the responses.

**3. Differentiate Between Outputs.**
- Learn how to identify the differences in outputs from each model.
- Recognize the strengths and weaknesses of each approach.
- Develop a clear rule of thumb for when to use each model based on the type of question you have.

**Practical Applications.** Real-world use: apply these models in practical scenarios to improve decision-making and problem-solving.

### The three Gemini modes

- **Fast.** Quick conversational answer. Good for *"what is X?"*, *"who founded Y?"*, latest news.
- **Thinking.** Spends time reasoning step-by-step. Good for *"compare X and Y on these 4 axes"*, multi-step logic, math.
- **Deep Research.** Browses 30+ sources for 10–30 min and writes a multi-page cited report. Good for *"market analysis"*, *"literature review"*, *"competitive landscape"*.

## 🧪 Lab: Same question, three Gemini modes

1. Open **Google Gemini** and pick your **one real question** from prep.
2. Run it in **Fast** mode. Note time and the key claim.
3. Run it in **Thinking** mode. Note time + the reasoning trace Gemini shows.
4. Run it in **Deep Research**. Walk away for 10–15 min while it works.
5. Re-prompt at least once using a Day 3 technique (CREATE / role / constraints) and observe how the output changes.
6. Build a 3-row table: mode · time · source quality · how confident you'd be quoting it in a viva.
7. Write 3 sentences: *for this question I would ship with __ because __.*

**Artifact.** Google Doc with the question, 3 outputs, comparison table, recommendation. Share link in the cohort channel.
