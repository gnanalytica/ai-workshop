> 🧠 **Quick glossary.** **Workflow** = a chain of boxes (nodes) that pass data left to right. **Node** = one box that does one job (scrape a page, ask an AI, send a message). **Trigger** = the first node — what starts the workflow (a button, a webhook, a schedule). **API key** = a long password that lets one service talk to another. **Scrape** = fetch a webpage and turn it into clean text.

## 🧪 Lab: Your first n8n workflow

Don't read ahead. We do this step-by-step in class, together. **If you get stuck, raise your hand — don't move on without help.**

### Step 1 — Create a new workflow

1. Open your n8n.cloud workspace.
2. Top-right corner: click **+ Create Workflow**.
3. You'll see a big empty canvas with a small **+** in the middle. That's where nodes go.
4. Rename the workflow (top of the screen) to: **"My first scraper"**.

### Step 2 — Add the trigger (the button)

1. Click the **+** in the middle of the canvas → search for **"Manual"** → click **Trigger manually**.
2. Close the side panel.

You should now see one orange node labelled **"When clicking 'Test workflow'"**. That's your button.

### Step 3 — Add Firecrawl (scrape the page)

1. Click the **+** to the right of the trigger node → search for **"HTTP Request"** → click it.
2. A side panel opens. Fill in these exact fields:

   - **Method:** `POST`
   - **URL:** `https://api.firecrawl.dev/v1/scrape`
   - **Authentication:** select `Generic Credential Type` → `Header Auth`.
     - Click **Create New Credential**.
     - **Header Name:** `Authorization`
     - **Header Value:** `Bearer fc-YOUR_KEY_HERE` (paste your Firecrawl key after the word `Bearer ` — keep the space).
     - Click **Save** → close the credential panel.
   - **Send Body:** toggle **ON**.
   - **Body Content Type:** `JSON`.
   - **Specify Body:** select `Using JSON`.
   - **JSON paste box:**
     ```json
     {
       "url": "https://en.wikipedia.org/wiki/Bengaluru",
       "formats": ["markdown"]
     }
     ```
     (Replace the Wikipedia URL with the one you brought to class if you like.)

3. Click **Test step** (top-right of the panel). Wait 5–10 seconds. If it worked you'll see a **green checkmark** and a big blob of markdown text below — that's the scraped page.

> If you see a red error, double-check the Bearer key. The single most common mistake: missing the space between `Bearer` and your key.

### Step 4 — Add Groq (the AI summarizer)

1. Click the **+** to the right of the HTTP Request node → search for **"HTTP Request"** again → click it.
2. Fill in these fields:

   - **Method:** `POST`
   - **URL:** `https://api.groq.com/openai/v1/chat/completions`
   - **Authentication:** select `Generic Credential Type` → `Header Auth`.
     - Click **Create New Credential**.
     - **Header Name:** `Authorization`
     - **Header Value:** `Bearer gsk_YOUR_KEY_HERE` (paste your Groq key after the word `Bearer ` — keep the space).
     - Click **Save** → close the credential panel.
   - **Send Body:** toggle **ON**.
   - **Body Content Type:** `JSON`.
   - **Specify Body:** `Using JSON`.
   - **JSON paste box:**
     ```json
     {
       "model": "llama-3.3-70b-versatile",
       "messages": [
         {
           "role": "user",
           "content": "Summarize this article in 5 bullet points. Be specific and use plain English. Mention any numbers, names, or dates that appear.\n\nARTICLE:\n{{ $json.data.markdown }}"
         }
       ]
     }
     ```

   The bit `{{ $json.data.markdown }}` is **n8n's way of saying "use the markdown from the previous node."** Type it exactly — the double curly braces and the spaces matter.

3. Click **Test step**. Groq is *fast* — usually under 2 seconds. You should see Groq's response in the bottom panel — find the field `choices → 0 → message → content`, which is your 5-bullet summary.

### Step 5 — Run the whole thing end-to-end

1. Top-right of the canvas: click **Test workflow**.
2. Watch the three nodes light up in order — first the trigger, then the scraper (green check), then Groq (green check).
3. Click on the Groq node → look at the right panel → expand `choices → 0 → message → content`. That's your 5-bullet summary.

If all three nodes are green, **you just shipped your first automation.** 🎉

### Step 6 — Change the URL and run again

Edit the Firecrawl node's URL to a different webpage. Click **Test workflow** again. Groq summarizes a totally different article in seconds.

**Artifact.** Take a screenshot showing all three nodes with green checkmarks **and** Groq's 5-bullet output. Submit on the dashboard via the assignment block below — paste the screenshot link, or paste the bullets straight into the text box.

> ⚠️ **API key safety.** Don't share the workflow with anyone — your keys are inside it. If you accidentally exposed a key, regenerate it from each service's dashboard (Firecrawl → API Keys → Revoke; Groq → API Keys → Delete).
