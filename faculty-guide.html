Support Faculty

Handbook

Your one-stop guide for supporting a cohort session — reference before every class.

Section 1

Your role in one paragraph

As Support Faculty you own three things inside a cohort: pod support (sitting with your assigned pod, answering questions, keeping energy high), attendance marking for your pod's students, and grading your pod's submissions against the Trainer-supplied rubric. What you don't own: curriculum content, cohort-wide decisions, platform configuration, or the final grade if you're unsure — that always escalates to the Trainer. Think of yourself as the closest expert to the learner, with a clear escalation path for anything outside your lane.
Section 2

The day of a session

  • Arrive 15 minutes early. Open the Faculty hub, select today's cohort, and confirm your pod list under People. Flag any unexpected absences to the Trainer before the session starts.
  • Mark attendance. Head to Attendance, find your pod members and mark each as present or absent as students arrive. Do this before the session content begins.
  • Sit with your pod. Position yourself physically (or in a breakout room) close to your pod so you can spot blocked body language quickly. Keep the Faculty hub open on a second screen or tab.
  • Triage questions in real time. Use the flowchart in Section 3 to decide: answer it yourself, Slack the Trainer, or escalate to Tech Support. Aim to unblock within 3–5 minutes; log anything that takes longer in the Stuck queue.
  • Close out the session. Check that all pod members have submitted their lab or marked it in-progress. Add a brief note in the Stuck queue for any issues that need Trainer follow-up. Sign off for the day in the pre-session checklist once all items are done.

Section 4

Common debugging recipes

Ollama won't start

Symptom

Running ollama serve or ollama run llama3 returns a connection refused error or the process exits immediately.

  1. Check Ollama is installed: ollama --version. If missing, reinstall from https://ollama.com/download.
  2. Kill any stuck process: pkill -f ollama (macOS/Linux) or end ollama.exe in Task Manager (Windows), then restart.
  3. Check available RAM — Ollama needs at least 8 GB free. Close heavy apps (browser, VS Code with many extensions) and retry.
If still stuck — escalate to Tech Support with: OS + version (e.g. macOS 14.4), Ollama version (ollama --version), full error text from terminal, and a screenshot of the terminal output.
Python venv broken

Symptom

import fails with ModuleNotFoundError even after running the install script, or python resolves to the wrong version.

  1. Confirm the venv is activated: terminal prompt should show (venv) or the env name. If not, run source venv/bin/activate (macOS/Linux) or venv\Scripts\activate (Windows).
  2. Reinstall dependencies inside the active venv: pip install -r requirements.txt. Check for error lines in the output.
  3. If Python version mismatch (python --version is not 3.11+), recreate venv with the correct interpreter: python3.11 -m venv venv.
If still stuck — escalate to Tech Support with: OS, Python version (python --version), error text, and a screenshot of the terminal after activation attempt.
VS Code extension missing

Symptom

A required extension (Python, Pylance, Jupyter, etc.) is not found or IntelliSense is not working.

  1. Open the Extensions panel (Ctrl+Shift+X / Cmd+Shift+X), search by name, and click Install.
  2. After installing, reload the window: Ctrl+Shift+P → Developer: Reload Window.
  3. If the extension installs but Python still isn't detected, open Command Palette → Python: Select Interpreter and choose the venv Python path.
If still stuck — escalate to Tech Support with: OS, VS Code version (Help → About), extension name and version, and a screenshot of the error or extension panel.
Supabase login loop

Symptom

Student clicks "Send magic link", receives the email, clicks the link, but is returned to the sign-in page in a loop.

  1. Clear browser cookies and site data for the workshop domain, then try the magic link again in a fresh tab.
  2. Try a different browser or an incognito/private window — some browsers block third-party storage required by Supabase Auth.
  3. Confirm the magic link hasn't expired (links expire after 1 hour). Resend a new magic link from the sign-in page.
If still stuck — escalate to Tech Support with: OS, browser name and version, description of the loop (e.g. "redirects back to login after clicking link"), and a screenshot of the URL in the address bar after clicking the link.
Magic link not arriving

Symptom

Student submitted their email but receives no magic link email after several minutes.

  1. Check spam/junk folder — email clients frequently filter transactional mail from Supabase.
  2. Confirm the email address is correct and matches the registration exactly (case-insensitive, but no typos).
  3. If nothing arrives within 5 minutes, ask the student to try again — there is a rate limit of one email per 60 seconds per address.
If still stuck — escalate to Tech Support with: OS, email client, email address (without sharing publicly), time the request was made, and a screenshot of their inbox/spam folder showing no email.
Section 5

Grading your pod

Open the day page (e.g. day.html?day=3), scroll down to the Submissions panel, and click a student's row to open their work. The rubric criteria are shown inline — score each dimension and leave a short, actionable comment explaining your score. Hit Save; the student's view updates immediately.

Stick to the rubric. If a submission is borderline or the rubric doesn't cover the situation, flag it to the Trainer rather than guess. Leave a note in the comment field: "Flagged — awaiting Trainer review." Never inflate grades to be kind — students rely on accurate feedback to improve.

Grades you submit are visible to students right away. If you need to revise after saving, re-open the row and update — revisions are tracked. All grading is scoped to your pod; you cannot see or edit other pods' submissions unless the Trainer explicitly grants access.
