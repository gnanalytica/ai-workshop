Lab Setup Guide

Step-by-step install playbook for preparing college lab computers before a cohort starts. Follow this guide once per machine — students do not install anything on their own during the workshop.

1. Before you start

You need admin / administrator rights on every lab machine. Without elevated privileges, several installers will silently fail. Confirm with your IT department before booking the setup slot.

Prereqs at a glance

  • Admin rights on each lab machine (local admin or domain admin)
  • Internet access from each lab machine — see the allowlist in System requirements
  • Cursor access for all faculty/student machines (primary IDE in this workshop)
  • ~30 min per machine for downloads + installs (faster on campus gigabit)
  • A USB drive or shared network folder with the installers pre-downloaded saves time if campus bandwidth is slow

Lab-wide install vs per-student install

Install all tools under a system-wide / all-users context whenever the installer allows it. This means every Windows user profile on the machine sees the same PATH entries and VS Code extensions. Do not install in "current user only" mode on shared lab machines.

Students do not install software on their own machines during the workshop. All tools should be present and verified on lab machines before Day 1. If a student needs to work from a personal device (laptop), point them to this guide and ask them to complete steps 1–5 at home before the cohort starts.

Ollama models are large (1–4 GB each). Pull them overnight on lab Wi-Fi or wired ethernet — not on the morning of Day 1 in front of students.

1A. One-time full setup (install everything upfront)

If you want everything ready at the beginning, complete this once before the cohort starts. This front-loads tools, accounts, keys, and verification so sessions stay focused on teaching.

Account & API key preflight (before Day 16)

Most delivery delays are account/auth issues, not installs. Complete these once per faculty account (and student cohort where relevant) before advanced days begin.

  • Required accounts: GitHub, Google account, Groq, Google AI Studio, HuggingFace, Vercel, Supabase, n8n.cloud.
  • Recommended accounts: Langfuse, Neon, LM Arena, Artificial Analysis, NotebookLM.
  • API keys ready: GEMINI_API_KEY (required), GROQ_API_KEY (required for some labs), Langfuse keys (optional).

Quick key check (Terminal)

# macOS/Linux
echo "$GEMINI_API_KEY" | wc -c
echo "$GROQ_API_KEY" | wc -c

# Windows PowerShell
$env:GEMINI_API_KEY.Length
$env:GROQ_API_KEY.Length

Python 3.11+ Required

Python 3.11 or 3.12 is required. The venv module must be available (it ships with the standard library on all platforms). Do not use Conda or Anaconda unless your cohort curriculum specifically requires it.

Download

https://www.python.org/ftp/python/3.11.9/python-3.11.9-amd64.exe

Verify

python -m venv --help | head -1   # Should print usage line
Common errors
python opens the Windows Store instead of Python
Go to Settings → Apps → Advanced app settings → App execution aliases and disable the Python Store aliases.

On Windows, the binary is python.exe. Create an alias: doskey python3=python $* or use python consistently in all workshop scripts.
pip install fails with SSL error
Corporate proxy is intercepting TLS. Ask IT for the proxy CA cert, then: pip install --cert path\to\ca.crt <package>.

Ollama Required

Ollama runs local language models. After installing the runtime, pull these two models — do this overnight on lab Wi-Fi, not on the morning of Day 1:

  • llama3.2:3b — ~2.0 GB download — primary workshop model
  • nomic-embed-text — ~274 MB download — embedding model for RAG labs
Bandwidth warning. Each model pull is 0.3–2 GB. Schedule pulls overnight or during lunch on a wired connection. Do not pull on mobile hotspot.
Download

https://ollama.com/download/OllamaSetup.exe

Install

Run OllamaSetup.exe as Administrator. Ollama installs as a background service.

Pull models (overnight)
ollama pull llama3.2:3b
ollama pull nomic-embed-text
Verify
ollama --version
# Expected: ollama version 0.5.x or newer
ollama list
# Should show llama3.2:3b and nomic-embed-text
Common errors
ollama: command not found
Reopen PowerShell after install. Ollama adds itself to the system PATH but existing sessions don't refresh.
Pull fails: Error: pull model manifest: Get registry.ollama.ai: dial tcp: i/o timeout
Campus firewall is blocking registry.ollama.ai. Ask IT to allowlist it, or pre-download models on a hotspot and copy the %USERPROFILE%\.ollama\models folder to each lab machine.
Model runs but inference is very slow (>60 s per response)
Machine has only 8 GB RAM and the model is larger than available memory. Swap to llama3.2:3b (already the smallest recommended). Close all browser tabs before running.

5. Escalation

If verification fails and the common-errors section above doesn't resolve the issue, reach Gnanalytica Tech Support with the following information:

  • OS and version (e.g. "Windows 11 22H2", "Ubuntu 22.04.4 LTS")
  • Which tool failed (e.g. "Ollama — step 3, pull command")
  • The exact error text — copy-paste or screenshot from the terminal
  • A screenshot of the terminal window showing the error

Primary channel — Community Board

Post to board.html with the tag setup. This creates a searchable record and allows the full support team to respond. Other Support Faculty may have solved the same issue.

Same-day blockers

For issues that block a session starting in less than 2 hours, use your cohort Slack or WhatsApp group directly. Ping the trainer first — they have escalation paths to Gnanalytica infrastructure.
